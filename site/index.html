This describes the resources that make up the BlockCypher API. If you have any problem or request please contact us.

Introduction

The BlockCypher API is a simple JSON API that can be accessed over HTTP or HTTPS from the api.blockcypher.com domain. It tries to adhere to the REST principles as much as possible and all URLs are versioned. In addition to straight HTTP APIs we also provide push-style APIs using websockets and web hooks. Web hooks let you register a URL that we will POST to anytime an event you're interested in happens.

Most of our resources exist under a given blockchain resource. For example the complete URL to the Bitcoin main blockcain is:

https://api.blockcypher.com/v1/btc/main

Opening that URL in your browser will produce the following JSON document, showing the current height of the chain as well as the hashes and URLs of that last 2 blocks (note that this is actually extracted in real time from our API).



  
The examples and samples in this documentation are in Javascript (browser or node.js) or using the command line. More samples will be added over time and complete examples can be fund in our Github documentation repository. For the command line, we recommend using curl and json. While installing them isn't required to get started, we will be using them in this documentation to illustrate some usage patterns. Like this little piece of magic showing the number of transactions in the latest block (see the Blockchain API section for more information):

  
$ curl -s http://api.blockcypher.com/v1/btc/main | json latestUrl | xargs curl -s | json n_tx
267

  

  
Blockchain API

  
The Blockchain API gives you access to the raw information that makes up the blockchain: blocks and transactions. If those concepts aren't familiar, refer to the Bitcoin wiki for more information about transactions and blocks.
  
Three main resources make up the Blockchain API:

  

    
NameURL PatternExampleDescription

    
    

      Chain/v1/{coin}/{chain}https://api.blockcypher.com/v1/btc/main
      Returns the current state of the specified blockchain, like the two latest blocks.
    

    

      Block/v1/{coin}/{chain}/blocks/{hash|height}
      https://api.blockcypher.com/v1/btc/main/blocks/293000
      All the details about a given block, including transaction hashes. Can be retrieved either by block height or block hash.
    

    

      Transaction/v1/{coin}/{chain}/txs/{hash}
      https://api.blockcypher.com/v1/btc/main/txs/f854aeba...
      All the details about a given transaction, including involved addresses, inputs and outputs.
