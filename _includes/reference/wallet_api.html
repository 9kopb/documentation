Wallet API

The wallet API groups multiple addresses under a single name. Addresses can be added and removed, and the name can be used with our other APIs. We also support HD (BIP 32) wallets. While for normal wallets any address can be added/removed, a HD wallet handles addresses derived from a single pair of master keys.

yes
Name URL Pattern Method Description
Create Wallet /v1/{coin}/{chain}/wallet/ (/v1/btc/main/wallet/) POST Creates a new standard wallet identified by name and associates addresses with it.
Create HD Wallet /v1/{coin}/{chain}/wallet/hd (/v1/btc/main/wallet/hd) POST Creates a new HD wallet. If no parameters are sent, a new wallet is generated and returned. See example below for return values. If you have already generated a HD wallet yourself, you can add it here by sending public_parent_key and chain_code.
Generate Address /v1/{coin}/{chain}/wallet/{name}/addresses/generate (/v1/btc/main/wallet/alice/addresses/generate) POST For a HD wallet, a new address is derived from the public parent key. For a non-HD wallet, a new address is generated similar to Address Generation and associated it with the given wallet.
Addresses /v1/{coin}/{chain}/wallet/{name}/addresses (/v1/btc/main/wallet/alice/addresses) GET All addresses in a given wallet.
Add Addresses /v1/{coin}/{chain}/wallet/{name}/addresses/ (/v1/btc/main/wallet/alice/addresses) POST Associates addresses with the wallet.
Remove Addresses /v1/{coin}/{chain}/wallet/{name}/addresss/ (/v1/btc/main/wallet/alice/addreses) DELETE addresses will no longer be associated with the wallet.
Delete /v1/{coin}/{chain}/wallet/{name}/ (/v1/btc/main/wallet/alice/) DELETEDeletes the wallet (removes the association between name and any addresses).

Wallet

To create a new standard wallet:

POST https://api.blockcypher.com/v1/btc/main/wallet/
{
  "name": "alice",
  "addresses": ["16uKw7GsQSzfMaVTcT7tpFQkd7Rh9qcXWX", "17astdTmG8zzVmry8mV8A7atAr3XefEgRX"]
}

HD wallet

To generate a new HD wallet:

POST https://api.blockcypher.com/v1/btc/main/wallet/hd
{
  "name": "alice",
}
This will return the private parent key, the public parent key and the chain code. See BIP 32. 
{
  "private_parent_key": "142ef411103ade32682cde5ded3a9085200c5efe65f1616d573d0d7fda37b1d9",
  "public_parent_key": "cd11ec8c9764b02b398b6cf772900fc52dace5db63d888e372a65baf3546dd44",
  "chain_code": "8b62b4a852ab78e8d4c07d5eae40de315f9d7446321b19a20f59aef083330aa9"
}

If you want to generate the HD wallet yourself, you can add it by sending the non-private parts. This provides the same functionality as if blockcypher generated the wallet, as we do not keep any private keys.

POST https://api.blockcypher.com/v1/btc/main/wallet/hd
{
  "name": "alice",
  "public_parent_key": "cd11ec8c9764b02b398b6cf772900fc52dace5db63d888e372a65baf3546dd44",
  "chain_code": "8b62b4a852ab78e8d4c07d5eae40de315f9d7446321b19a20f59aef083330aa9"
}
